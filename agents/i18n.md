# Localization (i18n) Agent

How translation works in this repo, how to add/translate keys, and how to audit for missing keys & English fallback leaks. Read this before touching any user-facing string.

## Files

| Path | What |
|------|------|
| `Quartz/Resource/Export/Lang/en-US.json` | English source. Shape: `{"en-US": { "KEY": "value", ... }}` |
| `Quartz/Resource/Export/Lang/ko-KR.json` | Korean. Shape: `{"ko-KR": { ... }}` |
| `Quartz/Resource/Export/Lang/zh-CN.json` | Chinese. **Translator-authored — don't hand-edit here** (see below). |
| `Quartz/Resource/Export/Lang/KEY_PREFIXES.md` | Maps opaque key prefixes (`AD_`, `DL_`, `FXRM_`, `JR_`, `JPOP_`, `KCB_`, `KL_`, `PCOL_`, `TW_`, `UIH_`) to owning features. |
| `Quartz/Localization/Translator.cs` | Loader + lookup (`Get`, `GetArr`, `GetForLanguage`, `HasKey`, `GetArrCount`). |
| `Quartz/Localization/TextLocalization.cs` | MonoBehaviour that re-renders a TMP label on language change. |
| `Quartz/UI/Generator/GenerateUI.cs` | Builders + key-derivation helpers (`LocalizeById`, `LocaleKeyFromId`, `LocaleKeyFromText`, `NormalizeLocaleKey`). |
| `scripts/validate_i18n.py` | Shape / `0KTL` / parity validator. Takes two positional args — see audit check 1. |
| `scripts/i18n_sync.py` | Sync with `PrismMods/Quartz-i18n`. `OWNED_HERE = {"en-US", "ko-KR"}`. |

- **`en-US` and `ko-KR` are authored here and must stay key-for-key identical** (same key set; only values differ). They're the two entries in `OWNED_HERE`.
- **Every other language is translator-authored** in [PrismMods/Quartz-i18n](https://github.com/PrismMods/Quartz-i18n) and is *allowed to lag* — `zh-CN` currently sits at 930 keys vs 1159, and `validate_i18n.py` only warns on its gaps. Don't hand-edit `zh-CN.json`; the next `i18n pull` overwrites you. See `agents/commits.md` for the push/pull cycle.
- **Key order is arbitrary.** Nothing sorts or normalizes these files — append a key wherever it's convenient.
- `0KTL` (`DO_NOT_TRANSLATE_THIS_KEY!`), `0NATIVELANG`, `0TRANSLATORS` are control entries — never translate `0KTL`. The loader **drops any language block whose `0KTL` doesn't match** (`Translator.cs`), so a translated `0KTL` silently kills that entire language.

## The core rule: missing key → English leak (no crash)

Every lookup takes an English **default**. If the key is absent (or the language is English/fallback), the default is returned. So a missing or wrong key is **silent** — it shows English in Korean, never an error. Auditing = finding those silent leaks.

## Key consumers (each needs its key present in BOTH files)

- `MainCore.Tr.Get(key, def)` / page-local `Tr(key, def)`
- `TextLocalization.Init(key, def)` (also via `GenerateUI.Localize(text, key, def)`)
- `AddToolTip(key, def)` — tooltip. **`AddToolTip(tip)` (one arg) is a raw literal, NOT localized.**
- `GetArr(key, index, def)` — array values (e.g. `0TRANSLATORS`)

## Keys derived from a builder `id` (the easy-to-miss path)

`Toggle`, `ToggleTip`, `Button`, `Slider`, `Input` (placeholder), `ColorPicker`, `KeyBind` auto-localize their label by calling `LocalizeById(label, id, text)`. The key is **`LocaleKeyFromId(id)`**, which is `NORM(id)` plus two rewrites:

```
NORM(s) = s.Trim().ToUpperInvariant(), every non-alphanumeric char -> '_',
          collapse repeats, strip TRAILING '_'   (leading '_' is kept; all-blank -> null)

LocaleKeyFromId(id, suffix?) = NORM(id), then:
          PANEL<digits>_X -> PANEL_X          (numeric infix after a leading PANEL is dropped)
          PANEL_PICK_X    -> PANEL_STAT_X
          + "_" + NORM(suffix)  when a suffix is passed
```

So `ToggleTip(..., "Boost Process Priority", "opt_priority", tip)` in `PageTweaks.cs` needs key **`OPT_PRIORITY`**. The English label arg is the fallback. Add a control with a new `id` → you owe a key.

- **`DropDown` does NOT localize its rows/selection by id** — those come from the `display` func, so localize there (see `DisplayFont` in `PageSettings.cs`). It **does** localize its `leftLabel` (when one is passed and `width > 0`) via `LocaleKeyFromId(id, "LABEL")` — so **`NORM(id) + "_LABEL"`** for ordinary ids, with the same `PANEL*` rewrites above. E.g. `AD_MODE_LABEL`, `KEYVIEWER_STYLE_LABEL`, `PANEL_ANCHOR_LABEL`. Adding a bare `*_DROPDOWN` key does nothing.
- An explicit `.Init(key,def)` on the label **overrides** the id-derived key (e.g. a button created with id `font_rename_btn` but re-localized via `.Init("FONT_RENAME", ...)` — both in `PageSettings.cs`).

## Dynamic key families (resolve at runtime — every value needs a key)

| Family | Built from | Source of values |
|--------|-----------|------------------|
| `PANEL_STAT_{ID}` | `PanelsOverlay.LocaleKey("PANEL_STAT_", stat.Id)` **and** `LocaleKeyFromText("PANEL_STAT", entry.Id)` in `PagePanels.cs` | `PanelsOverlay.Catalog` (StatDef list) |
| `PANEL_CATEGORY_{CAT}` | same two paths, `PANEL_CATEGORY` prefix | `Category` field on each StatDef |
| `SECTION_{TITLE}` | `LocaleKeyFromText("SECTION", title)` | each `GenerateUI.Collapsible(..., title)` |
| `ADDON_{TITLE}` | `LocaleKeyFromText("ADDON_", title)` | `AddonContext.cs` — addon-registered pages |
| `JPOP_*` | `NORM(id)` of the explicit `jpop_*` builder ids | `PageVisuals.PageBuilders.cs` |
| `JR_ALLOW_*` | hand-written `switch` on `HitMargin` | `Features/Restriction/Restriction.cs` |
| `PRESET_{NAME}` | `"PRESET_" + name.ToUpperInvariant()` (raw, no NORM) | preset files (runtime; default = name) |

**Neither `JPOP_*` nor `JR_ALLOW_*` tracks the enum name** — don't derive them mechanically. `HitMargin.FailMiss` → `JPOP_MISS` / `JR_ALLOW_MISS`; `HitMargin.OverPress` → `JPOP_OVERLOAD_FAIL`, and both `FailOverload` and `OverPress` map to the same `JR_ALLOW_OVERLOAD_FAIL`. Read the switch, don't guess.

Two different prefix helpers, and they behave differently: `PanelsOverlay.LocaleKey(prefix, id)` keeps the prefix **verbatim** and normalizes only the id; `GenerateUI.LocaleKeyFromText(prefix, text)` normalizes **both** and joins with `_`. To audit a family, enumerate its value set from the source and confirm each concrete key exists.

## How to add / translate a key

1. Pick the key name. For a builder control, it's `NORM(id)`; for a tooltip it's the `DESC_...` literal already in the call.
2. Add to **both** files with the same key. English = the in-code default verbatim; Korean = the translation.
3. Match existing terminology (grep ko for precedent):

   | English | Korean |
   |---|---|
   | Font / Font Size | 폰트 / 폰트 크기 |
   | Master Size | 전체 크기 |
   | Color / Text Color | 색상 / 글자 색상 |
   | Shadow / Shadow Softness / Shadow X | 그림자 / 그림자 부드러움 / 그림자 X |
   | Reset X Position | X 위치 초기화 |
   | + / - / X Perfect | + / - / X 퍼펙트 |
   | Enable X | X 활성화 |

4. **Keep English (do NOT translate):** acronyms (`BPM`, `CBPM`, `TBPM`, `FPS`, `KPS`, `MAX`, `X`), color channels `R/G/B/A`, the brand `Quartz`, the font sentinel `Default (Cookie Run Bold)` value (it's an identity/stored value — localize its *display* only, via the `FONT_DEFAULT` key + display func).
5. Preserve placeholders exactly: `{0}`, `{artist}`, `{title}`, `{n}`, and TMP rich-text tags `<color=...>`, `<size=...>`, `\n`.

### Bulk add / edit safely (Python — JSON round-trips exactly)

```python
import json, collections
path="Quartz/Resource/Export/Lang/ko-KR.json"; lang="ko-KR"
obj=json.load(open(path,encoding="utf-8"),object_pairs_hook=collections.OrderedDict)
obj[lang]["NEW_KEY"]="번역"
open(path,"w",encoding="utf-8").write(json.dumps(obj,ensure_ascii=False,indent=2)+"\n")
```

Always `ensure_ascii=False` (keep Hangul literal) and `indent=2`.

## How to apply / verify it compiles

JSON-only edits don't need a build — run `./test.sh` (it includes the en/ko parity test) or `python3 scripts/validate_i18n.py`.

After any **code** change (e.g. wrapping a hardcoded string):

```
./build.sh Debug ML          # or: dotnet build Quartz.slnx -c Debug
```

Must end `0 Error(s)`. Warnings are pre-existing and the count drifts — errors are the bar, not the warning count. Note the mod links the game's managed DLLs via `Directory.Build.props`, so this only works with a local ADOFAI install; CI builds and runs `Quartz.Tests` only.

## Audit checklist — run all of these

### 1. Key-for-key parity + JSON validity

Fastest path is the repo's own validator — it checks shape, `0KTL`, and parity in one shot. It takes `<lang_dir> <reference.json>`; there is no argparse, so a bare invocation just prints usage and exits 2:

```
python3 scripts/validate_i18n.py Quartz/Resource/Export/Lang Quartz/Resource/Export/Lang/en-US.json
```

It exits **1** on a missing or wrong `0KTL`, but only **warns** (exit 0) on translator-owned language gaps like `zh-CN`.

By hand:

```python
import json
en=set(json.load(open("Quartz/Resource/Export/Lang/en-US.json"))["en-US"])
ko=set(json.load(open("Quartz/Resource/Export/Lang/ko-KR.json"))["ko-KR"])
print("counts", len(en), len(ko))
print("in en not ko:", sorted(en-ko))
print("in ko not en:", sorted(ko-en))   # both must be empty
```

### 2. Every referenced literal key exists in both files

```python
import re, json, glob
en=set(json.load(open("Quartz/Resource/Export/Lang/en-US.json"))["en-US"])
ko=set(json.load(open("Quartz/Resource/Export/Lang/ko-KR.json"))["ko-KR"])
pat=re.compile(r'(?:\bGet|GetArr|GetForLanguage|HasKey|\.Init|AddToolTip|\bTr|\bLocalize)\(\s*"([A-Za-z0-9_]+)"')
refs={}
for f in glob.glob("Quartz/**/*.cs",recursive=True):
    if "/obj/" in f or "/bin/" in f: continue
    for m in pat.finditer(open(f,encoding="utf-8",errors="replace").read()):
        refs.setdefault(m.group(1), f)
miss=[(k,refs[k]) for k in refs if k not in en or k not in ko]
print("MISSING:", miss or "none")
```

Ignore a bare `DESC_` hit — that's the regex catching the literal half of `AddToolTip("DESC_" + id.ToUpperInvariant(), ...)` in `GenerateUI.cs`, not a real key.

### 3. Builder `id`-derived keys exist (catches unlocalized new controls)

For each `Toggle/ToggleTip/Button/Slider/Input/ColorPicker/KeyBind` call, compute `LocaleKeyFromId(id)` and check it's in both files. For each `DropDown` with a `leftLabel`, check `NORM(id) + "_LABEL"` too. Watch out:

- **Wrapper helpers pass `id` through** — e.g. `AddSlider` in `PageKeyViewer.Helpers.cs` (called from `PageKeyViewer.Tuning.cs`). Follow them to the real builder.
- Skip any label with an explicit `.Init(...)` override.
- The KeyViewer editor has its own builders that also call `LocalizeById` — `KvInspector.cs`, `KvWidgets.cs`, `KvWidgets.Input.cs`.

Easiest reliable pass: read each `Page*.cs` and resolve by hand, or fan out a subagent per file.

### 4. Dynamic families covered

Enumerate `PanelsOverlay.Catalog` stat Ids + categories, collapsible titles, addon page titles, and the `HitMargin` switch arms; confirm `PANEL_STAT_{ID}`, `PANEL_CATEGORY_{CAT}`, `SECTION_{TITLE}`, `ADDON_{TITLE}`, `JPOP_*`, `JR_ALLOW_*` all exist. Read the switch for the last two — the enum names don't match the keys.

### 5. Untranslated values (ko identical to en)

```python
import json
en=json.load(open("Quartz/Resource/Export/Lang/en-US.json"))["en-US"]
ko=json.load(open("Quartz/Resource/Export/Lang/ko-KR.json"))["ko-KR"]
acr={"BPM","CBPM","TBPM","FPS","KPS","MAX","X"}  # intentional
for k,v in en.items():
    if k in ("0KTL","0NATIVELANG"): continue
    if isinstance(v,str) and ko.get(k)==v and any(c.isalpha() for c in v) and v not in acr:
        print(k, repr(v))   # review each: translate or confirm intentional
```

### 6. Hardcoded literals (strings never routed through localization)

Grep `Page*.cs` and feature overlays for user-facing string literals assigned to `.text`, or passed to `AddToolTip(tip)` (one-arg), `FileBrowser`, etc., that have no key. Genuine user-facing ones → wrap in `MainCore.Tr.Get("KEY","English")` + add the key. Leave logs, exceptions, internal IDs/box-names, sentinels, single-letter glyphs, brand, and file-type filter labels.

## After an audit

- All six checks clean → done. State counts (`en=ko=N`; at time of writing, 1159).
- Wrapped a hardcoded string → `dotnet build` must pass.
- Commit per `agents/commits.md` (a localization-key add is `chore`; wrapping a string in `Tr.Get` is `feat`/`fix` by intent).
